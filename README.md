## 工作方式

- 先判断真实问题、用户角色、边界和不做什么，再扩产品范围。
- 用证据替代口号：区分原型、分支状态、主干状态、本地运行和已发布状态。
- 偏好高信息密度的 B 端界面：桌面优先、操作清晰、少装饰、可反复使用。
- 重视交付闭环：PRD / SSOT、验收标准、测试门槛、发布说明和残余风险都是产品的一部分。
